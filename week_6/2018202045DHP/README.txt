（打分类目标）
1.面向对象以及C++语法特征使用的正确性
    类内成员必然不能被直接修改，但可以通过某些成员函数进行值访问（非引用）。
    命名规则应该是基本遵守的（根据我的理解x）
    正常写的话风格应该不成太大问题。
    部分东西写死以方便将来的搜索引擎（？）的使用。
    
2.使用合适的STL容器和算法
    queue 和 set 在程序中广泛使用。
    爬虫使用广度优先搜索算法。

3.代码的规范性和可读性
    这个可以和1合在一起叭。
    英文命名完整清晰，应该比较好读。

4.加入异常和错误处理，日志记录，支持断点重爬
    丢失 personalsettings.ini 和 backup.ini 会报错，并自然退出程序。
    由于错误的网站导致wget功能失效会被计入错误日志。
    由于wget的功能错误或者网站无法访问将不被计入错误日志。
    正确日志的最后将记录时间消耗与爬取网站数量。
    断点重爬功能：每爬取10个网站将进行一次 等待队列与判重树的备份。
    注意，断点重爬时，请保持 personalsettings.ini 与原来保持一致，并备份result.out（该文件将被清空重新写入）内容。

5.加入命令行选项和功能控制等
    我比较希望客户将控制参数写入 personalsettings.ini
    命令行参数比较简单，可以写入 continue 以控制进行断点重爬

使用须知。
make
./IW 直接爬取
./IW continue 断点重爬

personalsettings.ini 提供三行操作。
line1: 根网站，并请在最后添加 / 符号
line2: 使用正则表达式ban掉需要去除的后缀名
line3: wget 参数
多余的行将被省略，不足的行将导致程序错误。
若程序异常终止，请不要修改 backup.ini
